<a href="index.html" class="body"></a><h2 class="first"><a name="id47471">Notes on RELAX NG Schema Version</a></h2>
<p class="first">The RELAX NG Schema has been generated from the <a href="index.html" class="body">Archiving and Interchange DTD</a> for use in RELAX NG tools.<b>The Schema is not intended for maintenance.</b></p>
<p>While the structural constraints on document tagging expressed by this schema are identical to those of the DTD, changes to the schema are best made by making the analogous changes to the DTD, and regenerating a schema from the modified DTD.</p>
<h2><a name="id47499">Getting the Files</a></h2>
<p class="first">The RELAX NG Schema files are available by anonymous FTP: <a href="ftp://ftp.ncbi.nih.gov/pub/archive_dtd/archiving/" class="body">ftp://ftp.ncbi.nih.gov/pub/archive_dtd/archiving/</a>.</p>
<p>A direct link to the files is available: <a href="ftp://ftp.ncbi.nih.gov/pub/archive_dtd/archiving/3.0/archive-interchange-rng-3.0.zip" class="body">ftp://ftp.ncbi.nih.gov/pub/archive_dtd/archiving/3.0/archive-interchange-rng-3.0.zip</a>.</p>
<p>The RELAX NG Schema is available on the web: <a href="rng/archivearticle3.rng" class="body">http://dtd.nlm.nih.gov/archiving/3.0/rng/archivearticle3.rng</a></p>
<h2><a name="id47556">Schema Structure</a></h2>
<p class="first">The main module of the RELAX NG schema is journalpublishing.rng. It imports the subordinate modules. For the schema to work all modules must be in the same directory.</p>
<h2><a name="id47567">Tools</a></h2>
<p class="first">The Schema has been tested and found to perform properly in James Clark's Jing Validator running in syncROsoft &lt;oXygen/&gt; (versions 9 and 9.2).</p>
