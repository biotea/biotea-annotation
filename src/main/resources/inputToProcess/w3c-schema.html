<a href="index.html" class="body"></a><h2 class="first"><a name="id47441">Notes on W3C XML Schema Version</a></h2>
<p class="first">The W3C XML Schema has been generated from the <a href="index.html" class="body">Archiving and Interchange DTD</a> for use in XML tools that can not read DTDs.<b>The Schema is not intended for maintenance.</b></p>
<p> In order to mitigate problems found in testing with various processors' differing interpretations of Schema syntax, this Schema has been generated out of a flattened rendition of the NLM Archiving and Interchange DTD. While the structural constraints on document tagging expressed by this schema are identical to those of the DTD, the DTD's modular architecture is not reflected in the schema's organization. Changes to the Tag Set will be made by making modifications to the DTD, and a modified Schema will be generated from the modified DTD.</p>
<h2><a name="id47472">Getting the Files</a></h2>
<p class="first">The W3C XML Schema files are available by anonymous FTP: <a href="ftp://ftp.ncbi.nih.gov/pub/archive_dtd/archiving/" class="body">ftp://ftp.ncbi.nih.gov/pub/archive_dtd/archiving/</a></p>
<p>A direct link to the files is available: <a href="ftp://ftp.ncbi.nih.gov/pub/archive_dtd/archiving/3.0/archive-interchange-xsd-3.0.zip" class="body">ftp://ftp.ncbi.nih.gov/pub/archive_dtd/archiving/3.0/archive-interchange-xsd-3.0.zip</a></p>
<p>The W3C XML Schema is available on the web: <a href="xsd/archivearticle3.xsd" class="body">http://dtd.nlm.nih.gov/archiving/3.0/xsd/archivearticle3.xsd</a></p>
<h2><a name="id47527">Schema Structure</a></h2>
<p class="first">The Schema contains four modules:
			<ul>
<li class="ul"><p>archivearticle3.xsd</p></li>
<li class="ul"><p>xlink.xsd</p></li>
<li class="ul"><p>xml.xsd</p></li>
</ul></p>
<p>In addition, a subdirectory contains a copy of MathML (see <a href="http://www.w3.org/Math/XMLSchema/" class="body">http://www.w3.org/Math/XMLSchema/</a>), which provides additional modules.</p>
<p>Each of these modules corresponds to one of the namespaces in play:

			<ul>
<li class="ul"><p>archivearticle3.xsd (null, default namespace)</p></li>
<li class="ul"><p>xlink.xsd          (xlink namespace: http://www.w3.org/1999/xlink)</p></li>
<li class="ul"><p>xml.xsd            (xml namespace: http://www.w3.org/XML/1998/namespace)</p></li>
<li class="ul"><p>ncbi-mathml2/mathml2.xsd   (mathml namespace: http://www.w3.org/1998/Math/MathML)</p></li>
</ul></p>
<p>The modules import one another. For the schema to work all modules must
be in the same directory. </p>
<h2><a name="id47604">Tools</a></h2>
<p class="first">The Schema has been tested and found to perform properly in:
				<ol>
<li><p>&lt;oXygen/&gt; versions 9.0, 9.2</p></li>
<li><p>Xerces-J version 2.6.2</p></li>
<li><p>XML Spy (version 2004 rel.3)</p></li>
<li><p>Microsoft Word XML 2003 Professional (see below)</p></li>
</ol></p>
<p>Users of other tools may find anomalies due to discrepancies in various processors' differing interpretations of Schema syntax. In such cases, it is recommended to use the normative version of the NCBI Archive and Interchange DTD.</p>
<p>The document element of an instance document, depending on the tools used, may require namespace and attribute declarations to associate the document with the schema. For Xerces and XMLSpy, these may take the form of:

<blockquote>&lt;article xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:noNamespaceSchemaLocation="archivearticle3.xsd"&gt;</blockquote></p>
<p>where the xsi:noNamespaceSchemaLocation attribute points to the archivearticle3.xsd module on the system.</p>
<div class="level2">
<a name="id47659"></a>MS Word 2003</div>
<p class="first">Cautions when using MS Word 2003 Professional</p>
<p>The mechanism for associating a document with a schema in MS Word 2003 Professional differs considerably
from the above. The declarations (namespace declaration for the xsi namespace, and the
xsi:noNamespaceDeclaration attribute) are NOT required and may confuse the system.</p>
<p>A schema may be bound to a document in MS Word 2003 as follows:
			<ol>
<li>
<p>Add the schema to the list of schemas available. This can be done (though only when an XML document is open) through the "XML Schema" tab under the "Templates and Add-ins" menu option (under "Tools"). When the schema is selected, Word will want a URI to associate with it (despite the fact that this document type has no default namespace). We have used the following:
					<blockquote>URI: http://namespace.nih.gov/archivearticle3.xsd</blockquote><blockquote>Alias: Archive and Interchange schema</blockquote></p>
<p>On adding this namespace, you will be asked also to identify the modules

<blockquote><b>XLink namespace</b></blockquote><blockquote>URI: http://www.w3.org/1999/xlink</blockquote><blockquote>Alias: W3C XLink namespace</blockquote><blockquote><b>XML namespace</b></blockquote><blockquote>URI: http://www.w3.org/XML/1998/namespace</blockquote><blockquote>Alias: W3C XML namespace</blockquote><blockquote><b>MathML</b></blockquote><blockquote>URI: http://www.w3.org/1998/Math/MathML</blockquote><blockquote>Alias: W3C MathML2 schema</blockquote></p>
</li>
<li>
<p>In the XML Schema tab of the Templates and Add-ins dialog box, associate the open document with the schema by
				checking the box next to the schema in the list.</p>
<p>Use the "Save As XML" feature with the "Save Data only" box checked to save clean XML using only allowed
				tags.</p>
</li>
</ol></p>
